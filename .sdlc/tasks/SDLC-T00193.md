## Objective

Enhance the existing `a-sdlc run status` command to display pipeline-specific columns (phase, queue depth, thread count) alongside existing sprint/task runs, and add a per-run detail view activated by passing a `run_id` argument. This surfaces pipeline observability directly in the CLI, covering requirements FR-009 (pipeline status in summary table), FR-010 (per-run detail view), and FR-010a (clarification visibility).

## Requirements Traced

| Requirement | Description | Coverage |
|---|---|---|
| **FR-009** | `run status` summary shows pipeline runs with phase and queue info | Enhanced summary table with Phase, Queue, Threads columns |
| **FR-010** | `run status <run_id>` shows detailed queue state and thread entries | New `_show_run_detail()` helper function |
| **FR-010a** | Clarification question visible when run is paused | Detail view renders clarification panel |
| **NFR-003** | Existing `run status` behavior unchanged for non-pipeline runs | Pipeline columns show "-" for sprint/task runs |

## Current State Analysis

### Existing Implementation: `src/a_sdlc/cli.py` lines 3844-3947

The current `run_status` command:

1. Takes no arguments (`@run.command("status")` / `def run_status() -> None`)
2. Reads all `R-*.json` files from `~/.a-sdlc/runs/` via `_read_run()` from `a_sdlc.executor`
3. Displays a Rich `Table` with columns: Run ID, Type, Entity, Status, PID, Started, Details
4. Checks PID liveness via `os.kill(pid, 0)` and marks dead processes as "crashed"
5. Shows approval instructions for runs with `status == "awaiting_confirmation"`

### Step 2: Extract existing logic into `_show_all_runs()`

Move the current `run_status` body (lines 3855-3947) into a new `_show_all_runs()` function, then add three pipeline-specific columns.

```python
def _show_all_runs() -> None:
    """Display summary table of all background runs."""
    import os

    from a_sdlc.executor import _RUNS_DIR, _read_run

    runs_dir = _RUNS_DIR
    if not runs_dir.exists():
        console.print("[yellow]No runs found.[/yellow]")
        return

    run_files = sorted(runs_dir.glob("R-*.json"), key=lambda p: p.stat().st_mtime, reverse=True)
    if not run_files:
        console.print("[yellow]No runs found.[/yellow]")
        return

    table = Table(title="Background Runs")
    table.add_column("Run ID", style="bold")
    table.add_column("Type")
    table.add_column("Entity")
    table.add_column("Status")
    table.add_column("Phase")         # NEW: pipeline phase
    table.add_column("Queue")         # NEW: pending/active/completed counts
    table.add_column("Threads")       # NEW: thread entry count
    table.add_column("PID")
    table.add_column("Started")
    table.add_column("Details")

    for run_file in run_files:
        file_run_id = run_file.stem
        data = _read_run(file_run_id)
        if not data:
            continue

        # Check if the process is actually still alive
        pid = data.get("pid")
        pid_alive = False
        if pid:
            try:
                os.kill(pid, 0)
                pid_alive = True
            except (OSError, ProcessLookupError):
                pid_alive = False

        status = data.get("status", "unknown")
        if status == "running" and pid and not pid_alive:
            status = "crashed"

        status_style = {
            "running": "[green]running[/green]",
            "completed": "[blue]completed[/blue]",
            "failed": "[red]failed[/red]",
            "cancelled": "[yellow]cancelled[/yellow]",
            "crashed": "[red]crashed[/red]",
            "awaiting_confirmation": "[magenta]awaiting approval[/magenta]",
            "awaiting_clarification": "[magenta]awaiting input[/magenta]",
        }.get(status, status)

        # Build details string
        details = ""
        if status == "awaiting_confirmation":
            config = data.get("config", {})
            details = config.get("message", "Waiting for approval")
        elif status == "awaiting_clarification":
            q = data.get("clarification_question", "")
            details = (q[:50] + "...") if len(q) > 50 else q
        elif data.get("supervised"):
            details = "supervised"

        started = data.get("started_at", "")
        if started:
            started = started[:19].replace("T", " ")

        # --- Pipeline-specific columns ---
        run_type = data.get("type", "?")
        phase = "-"
        queue_info = "-"
        thread_count = "-"

        if run_type == "pipeline":
            phase = data.get("phase", "-")
            work_queue = data.get("work_queue", [])
            if work_queue:
                counts = {"pending": 0, "active": 0, "completed": 0}
                for item in work_queue:
                    s = item.get("status", "pending")
                    if s in counts:
                        counts[s] += 1
                    else:
                        counts["pending"] += 1
                queue_info = f"{counts['pending']}p/{counts['active']}a/{counts['completed']}c"
            thread_entries = data.get("thread_entries", [])
            thread_count = str(len(thread_entries))

        table.add_row(
            file_run_id,
            run_type,
            data.get("entity_id", "?"),
            status_style,
            phase,
            queue_info,
            thread_count,
            str(pid) if pid else "-",
            started,
            details,
        )

    console.print(table)

    # Show approval instructions for any awaiting runs
    for run_file in run_files:
        file_run_id = run_file.stem
        data = _read_run(file_run_id)
        if data and data.get("status") == "awaiting_confirmation":
            console.print()
            console.print(f"[bold magenta]Run {file_run_id} is awaiting approval:[/bold magenta]")
            config = data.get("config", {})
            if config.get("message"):
                console.print(f"  {config['message']}")
            console.print(f"  Approve: [cyan]a-sdlc run approve {file_run_id}[/cyan]")
            console.print(f"  Reject:  [cyan]a-sdlc run reject {file_run_id} -m 'reason'[/cyan]")
```

### Step 3: Implement `_show_run_detail(run_id)`

New helper function for the per-run detail view.

```python
def _show_run_detail(run_id: str) -> None:
    """Display detailed state for a specific run."""
    from datetime import datetime, timezone

    from a_sdlc.executor import _read_run

    data = _read_run(run_id)
    if not data:
        console.print(f"[red]Run not found: {run_id}[/red]")
        sys.exit(1)

    run_type = data.get("type", "?")
    status = data.get("status", "unknown")
    entity_id = data.get("entity_id", "?")
    started = data.get("started_at", "")

    # --- Compute elapsed time ---
    elapsed = ""
    if started:
        try:
            start_dt = datetime.fromisoformat(started)
            now = datetime.now(timezone.utc)
            if start_dt.tzinfo is None:
                from datetime import timezone as tz
                start_dt = start_dt.replace(tzinfo=tz.utc)
            delta = now - start_dt
            minutes, seconds = divmod(int(delta.total_seconds()), 60)
            hours, minutes = divmod(minutes, 60)
            if hours > 0:
                elapsed = f"{hours}h {minutes}m"
            else:
                elapsed = f"{minutes}m {seconds}s"
        except (ValueError, TypeError):
            elapsed = "?"

    # --- Run info panel ---
    info_lines = [
        f"Run ID:    [bold]{run_id}[/bold]",
        f"Type:      {run_type}",
        f"Entity:    {entity_id}",
        f"Status:    {status}",
        f"Started:   {started[:19].replace('T', ' ') if started else '-'}",
        f"Elapsed:   {elapsed}",
    ]

    if run_type == "pipeline":
        info_lines.insert(3, f"Goal:      {data.get('goal', '-')}")
        info_lines.insert(4, f"Phase:     {data.get('phase', '-')}")

    if data.get("pid"):
        info_lines.append(f"PID:       {data['pid']}")

    console.print(Panel("\n".join(info_lines), title="[bold]Run Details[/bold]", border_style="blue"))

    # --- Work queue table (pipeline only) ---
    work_queue = data.get("work_queue", [])
    if work_queue:
        console.print()
        wq_table = Table(title="Work Queue")
        wq_table.add_column("ID", style="bold")
        wq_table.add_column("Type")
        wq_table.add_column("Artifact")
        wq_table.add_column("Status")
        wq_table.add_column("Persona")
        wq_table.add_column("Started")
        wq_table.add_column("Duration")

        for item in work_queue:
            item_started = item.get("started_at", "")
            item_completed = item.get("completed_at")
            duration = "-"
            if item_started and item_completed:
                try:
                    s = datetime.fromisoformat(item_started)
                    e = datetime.fromisoformat(item_completed)
                    dur_sec = int((e - s).total_seconds())
                    duration = f"{dur_sec}s"
                except (ValueError, TypeError):
                    duration = "?"
            elif item_started and item.get("status") == "active":
                duration = "running"

            item_status = item.get("status", "?")
            status_styled = {
                "pending": "[dim]pending[/dim]",
                "active": "[green]active[/green]",
                "completed": "[blue]completed[/blue]",
                "failed": "[red]failed[/red]",
            }.get(item_status, item_status)

            wq_table.add_row(
                item.get("id", "?"),
                item.get("type", "?"),
                item.get("artifact_id", "-"),
                status_styled,
                item.get("persona", "-"),
                item_started[:19].replace("T", " ") if item_started else "-",
                duration,
            )

        console.print(wq_table)

    # --- Recent thread entries ---
    thread_entries = data.get("thread_entries", [])
    if thread_entries:
        console.print()
        recent = thread_entries[-10:]  # Last 10 entries
        entry_lines = []
        for entry in recent:
            ts = entry.get("timestamp", "")[:19].replace("T", " ")
            phase = entry.get("phase", "")
            agent = entry.get("agent", "")
            action = entry.get("action", "")
            entry_lines.append(f"[dim]{ts}[/dim]  [{phase}] {agent}: {action}")
        console.print(
            Panel(
                "\n".join(entry_lines),
                title=f"[bold]Recent Activity ({len(thread_entries)} total)[/bold]",
                border_style="cyan",
            )
        )

    # --- Clarification panel ---
    clarification = data.get("clarification_question")
    if clarification:
        console.print()
        console.print(
            Panel(
                f"[yellow]{clarification}[/yellow]\n\n"
                f"Respond: [cyan]a-sdlc run respond {run_id} -m 'your answer'[/cyan]",
                title="[bold yellow]Clarification Needed[/bold yellow]",
                border_style="yellow",
            )
        )

    # --- Metrics summary (pipeline only) ---
    metrics = data.get("metrics", {})
    if metrics:
        console.print()
        metric_lines = []
        if metrics.get("total_duration_sec"):
            mins, secs = divmod(metrics["total_duration_sec"], 60)
            metric_lines.append(f"Total duration:    {int(mins)}m {int(secs)}s")
        if metrics.get("phase_durations"):
            phases = metrics["phase_durations"]
            phase_strs = [f"{k}: {v}s" for k, v in phases.items()]
            metric_lines.append(f"Phase durations:   {', '.join(phase_strs)}")
        if metrics.get("agent_session_count") is not None:
            metric_lines.append(f"Agent sessions:    {metrics['agent_session_count']}")
        if metrics.get("challenge_rounds") is not None:
            metric_lines.append(f"Challenge rounds:  {metrics['challenge_rounds']}")
        if metrics.get("failure_count") is not None:
            metric_lines.append(f"Failures:          {metrics['failure_count']}")
        if metrics.get("total_cost_cents") is not None:
            cost = metrics["total_cost_cents"] / 100
            metric_lines.append(f"Total cost:        ${cost:.2f}")
        if metrics.get("total_turns") is not None:
            metric_lines.append(f"Total turns:       {metrics['total_turns']}")

        if metric_lines:
            console.print(
                Panel(
                    "\n".join(metric_lines),
                    title="[bold]Metrics[/bold]",
                    border_style="green",
                )
            )
```

### Key Design Decisions

1. **Pipeline columns show "-" for non-pipeline runs**: The three new columns (Phase, Queue, Threads) display a dash for sprint/task runs. This is visually clean and avoids conditional column rendering that would make the table structure inconsistent.

2. **Queue info uses compact format `Xp/Ya/Zc`**: Pending/active/completed counts in a single cell. This is dense but scannable and avoids making the table too wide.

3. **`awaiting_clarification` as a new status style**: Pipeline runs can pause for user input, distinct from the existing `awaiting_confirmation` (supervised approval). Both render in magenta but with different labels ("awaiting input" vs "awaiting approval").

4. **Detail view uses Rich Panels and Tables**: Matches the existing CLI output patterns (e.g., `run sprint` and `run task` commands use `Panel` for output). The detail view composes multiple panels vertically for a dashboard-like experience.

5. **Thread entries capped at last 10**: Avoids flooding the terminal. The panel header shows total count so the user knows more exist.

6. **Elapsed time is computed live**: For running pipelines, the elapsed time is calculated from `started_at` to now, giving real-time duration.
