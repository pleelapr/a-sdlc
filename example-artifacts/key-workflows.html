Workflows

The following key workflows were identified based on code analysis, focusing on the LangGraph agent's adventure generation process. The codebase contains 20 distinct workflows due to its focused scope on generating Dungeons and Dragons adventures through a multi-agent LangGraph system. Workflows 1–15 cover the agent pipeline; the five utility workflows (file writing, format conversion, and cloud upload) are summarized in Utility Workflows.

1. Adventure Generation Orchestration

Orchestrates the entire adventure generation process, chaining various agents to produce a complete adventure document. Entry point: workflow.ainvoke in src/agent/graph.py receives an InputState (adventure description, difficulty, chapters) and each agent updates the shared OutlineState in turn.

Main components: graph.py, state.py, prompt_improver.py, editor_selector.py, interview_graph.py, outliner.py, table_of_content_agent.py, section_writer_agent.py, publisher_agent.py. Relevance: primary entry point (LangGraph workflow definition), multi-component orchestration, core domain focus.

2. Prompt Improvement

Refines an initial adventure idea into a more detailed and actionable prompt for subsequent generation steps.

Main components: prompt_improver.py, state.py, nodes.py.

graph.py
  → PromptImproverAgent.run: receives InputState
      → PromptImproverAgent.execute (self.prompt | self.model): invokes LLM with initial idea
      ← returns improved prompt string
  ← updates OutlineState.improved_prompt

3. Editor Selection and Subject Survey

Identifies related subjects for an improved prompt and generates personas for "editors" who will participate in the interview process. Critical external integration: WikipediaRetriever.

Main components: editor_selector.py, state.py, nodes.py, tools.py.

graph.py
  → EditorSelectorAgent.run: receives OutlineState with improved_prompt
      → survey_subjects (self.generate_idea): generates related topics using LLM
      → survey_subjects (WikipediaRetriever.abatch): retrieves documents from Wikipedia
      → generate_editor (model.with_structured_output(Perspectives)): generates editor personas
      ← returns a list of Editor objects
  ← updates OutlineState.editors

4. Interview Question Generation

Generates interview questions from the InterviewQuestionAgent based on the current interview state and editor persona.

Main components: interview_graph.py, state.py, nodes.py.

InterviewAgent.run_interview
  → InterviewQuestionAgent.generate_question: receives InterviewState
      → self.prompt | self.model: invokes LLM to generate a question
      ← returns an AIMessage containing the question
  ← updates InterviewState with the new message

5. Interview Answer Generation with Citations

Generates answers to interview questions, potentially using external search tools, and includes citations for the information. Critical external integration: TavilySearch.

Main components: interview_graph.py, state.py, nodes.py, tools.py.

InterviewAgent.run_interview
  → InterviewAnswerAgent.run: receives InterviewState with a question
      → gen_queries_chain: invokes LLM to generate search queries
      → discuss_engine (TavilySearch): executes search queries
      → gen_answer_chain (model.with_structured_output(AnswerWithCitations)): generates an answer with citations
      ← returns an AIMessage containing the answer and cited URLs
  ← updates InterviewState with the new message

6. Interview Summary Generation

Summarizes the entire interview log for a specific editor, extracting key information.

Main components: interview_graph.py, state.py, nodes.py.

graph.py
  → InterviewAgent.run_interview: after all interviews are complete
      → InterviewSummaryAgent.generate_summary (self.prompt | self.model): invokes LLM with the full interview log
      ← returns a summary string
  ← updates OutlineState.interview_summary

7. Initial Adventure Outline Drafting

Generates the high-level structure of the adventure, including chapters and their synopses, based on the improved prompt and interview summaries.

Main components: outliner.py, state.py, outline_schema.py, nodes.py, writer_agent.py.

graph.py
  → OutlinerAgent.run_initial_outliner: receives OutlineState
      → draft_outline (model.with_structured_output(AdventureOutline)): generates AdventureOutline
      → run_writing_outline_text (writer_agent.run_writing_text): converts the outline to markdown
      ← returns the structured outline and its markdown representation
  ← updates OutlineState.outline_data and OutlineState.outline_text

8. Table of Content Generation

Generates a detailed table of contents, including chapters, characters, and items, based on the initial adventure outline and interview summaries.

graph.py
  → TableOfContentAgent.run_writing_table_of_content: receives OutlineState
      → run_writing_table_of_content_detail (model.with_structured_output(TableOfContent)): generates TableOfContent
      ← returns the structured table of contents
  ← updates OutlineState.table_of_contents

SectionWriterAgent.run_writing_subsection_text_parallel

10. Section Outline Improvement

Takes a chapter outline and comments, then generates an improved version of the outline.

Main components: section_writer_agent.py, state.py, section_schema.py, nodes.py.

SectionWriterAgent.run_writing_subsection_text_parallel

11. Subsection Text Writing

Generates the detailed narrative content for a specific subsection of a chapter.

SectionWriterAgent.run_writing_subsection_text_parallel

      ← returns the generated text

12. Subsection Text Commenting

Reviews the generated text for a subsection and provides comments or recommendations for improvement.

Main components: section_writer_agent.py, state.py, nodes.py.

SectionWriterAgent.run_writing_subsection_text_parallel
  → SubSectionCommenterAgent.run_comment_subsection_text: receives SectionTextState with subsection text
      → self.prompt | self.model: invokes LLM to generate comments on the text
      ← returns a list of comment strings
  ← updates SectionTextState.comment

13. Statblock Content Writing

Generates the detailed statblock content for a character (monster or NPC).

Main components: section_writer_agent.py, statblock_schema.py, nodes.py, writer_agent.py.

SectionWriterAgent.run_writing_subsection_text_parallel
  → StatblockWriterAgent.run_writing_statblock_text: receives Character object and difficulty
      → writer_agent.run_writing_text: invokes LLM using MarkdownExampleKey.STATBLOCK
      ← returns the generated statblock text
  ← statblock text returned to the caller

14. Item Content Writing

Generates the detailed description and features for an item.

Main components: section_writer_agent.py, item_schema.py, nodes.py, writer_agent.py.

SectionWriterAgent.run_writing_subsection_text_parallel
  → ItemWriterAgent.run_writing_item_text: receives item name and description
      → writer_agent.run_writing_text: invokes LLM using MarkdownExampleKey.ITEM
      ← returns the generated item text
  ← item text returned to the caller

15. Full Document Content Assembly

Gathers all generated sections, statblocks, and items, and assembles them into a single, coherent markdown document.

Main components: publisher_agent.py, state.py.

graph.py
  → PublisherAgent.run: receives OutlineState containing all generated content
      → generate_full_content: iterates through section_texts, statblocks, and items
      → concatenates all content into a single markdown string
      ← returns the complete markdown layout string
  ← passes the layout string to write_report_by_formats

Utility Workflows

Five utility workflows in src/agent/utils/file_formats.py handle major data operations for any calling agent (typically the PublisherAgent):

WorkflowEntry PointFlow
16. Markdown File Writingwrite_text_to_mdEnsures directory exists (os.path.dirname), asynchronously writes text to a .md file via aiofiles.open, returns the file path.
17. JSON File Writingwrite_jsonEnsures directory exists, serializes a dictionary with json.dumps, asynchronously writes the JSON string to a .json file, returns the file path.
18. Markdown to PDF Conversionwrite_md_to_pdfConverts Markdown to HTML via mistune.html, converts HTML to PDF via an external tool, returns the encoded file path of the PDF.
19. Markdown to Word Conversionwrite_md_to_wordConverts Markdown to HTML via mistune.html, builds a DOCX with Document(), saves it with doc.save, returns the encoded file path.
20. Markdown to Azure Blob Uploadwrite_md_to_azure_blobCreates a BlobServiceClient from the connection string, ensures the container exists, uploads the text via blob_client.upload_blob, returns the blob URL. Critical external integration: Azure Blob Storage.
