Component Overview

This codebase implements a LangGraph-based agent system designed for generating Dungeons and Dragons 5E adventure content. The architecture is primarily agent-oriented, leveraging large language models (LLMs) for various content generation and refinement tasks, orchestrated through a stateful graph. The core technology stack includes Python, LangChain, LangGraph, Pydantic for data schemas, and OpenAI models, with deployment facilitated by Docker and integration with external services like Tavily Search and Azure Blob Storage.

src/agent/graph.py

Defines the overall LangGraph workflow for the Realmforge Agent, orchestrating the sequence and interaction of various specialized agents to generate adventure content. The primary export is workflow, a StateGraph instance that defines the nodes and edges connecting the different agents. It instantiates PromptImproverAgent, EditorSelectorAgent, InterviewAgent, OutlinerAgent, TableOfContentAgent, SectionWriterAgent, and PublisherAgent, manages the overall OutlineState as it flows through the graph, and defines the Configuration TypedDict for graph parameters.

src/agent/utils/state.py

Defines the TypedDict data structures that represent the state of the agent system at different stages: InputState (adventure_description, difficulty, number_of_chapters), OutlineState (a comprehensive set of fields such as improved_prompt, editors, outline_data, section_texts, statblocks, items, report, and url), and OutputState. This module is central to state management, explicitly defining the schema for the shared state passed between nodes in the LangGraph workflow.

src/agent/utils/prompt_improver.py

The PromptImproverAgent takes an initial, high-level adventure idea and refines it into a more detailed and actionable prompt. Its execute method refines the prompt via an LLM and ChatPromptTemplate; its run method reads adventure_description, number_of_chapters, and difficulty from InputState and updates improved_prompt in OutlineState.

src/agent/utils/editor_selector.py

The EditorSelectorAgent surveys related subjects based on the improved prompt and generates a list of "editors" (personas) with specific affiliations, roles, and descriptions. It uses a WikipediaRetriever for information gathering and LLM chains (survey_subjects, generate_idea, generate_editor) to produce Editor Pydantic objects, populating the editors field in OutlineState.

src/agent/utils/interview_graph.py

The InterviewAgent orchestrates a simulated interview where an InterviewQuestionAgent and an InterviewAnswerAgent converse to gather detailed information for each editor persona, with an InterviewSummaryAgent summarizing the logs. It builds a sub-graph using StateGraph for the question-answer loop and defines the InterviewState, AnswerWithCitations, AnswerWithoutCitations, and Queries models. It reads editors and improved_prompt from OutlineState and populates interview_result and interview_summary.

src/agent/utils/outliner.py

The OutlinerAgent generates the initial high-level adventure outline from the improved prompt and interview summaries. draft_outline produces a structured AdventureOutline via LLM structured output, and run_writing_outline_text converts it to markdown using an internal WriterAgent. It populates outline_data and outline_text in OutlineState.

src/agent/utils/table_of_content_agent.py

The TableOfContentAgent generates a detailed table of contents (chapters, characters, and items) from the initial outline and interview summaries, using an LLM with structured output to produce a TableOfContent Pydantic object stored in OutlineState.table_of_contents.

src/agent/utils/section_writer_agent.py

The SectionWriterAgent generates the detailed text content for individual sections of the adventure, often with iterative refinement. It orchestrates SubSectionWriterAgent, StatblockWriterAgent, and ItemWriterAgent (plus commenter/improver sub-agents) through sub-graphs, and exposes run_writing_subsection_text_parallel for concurrent writing tasks. It defines SectionImproverState and SectionTextState and produces section texts, statblocks, and item statblocks for OutlineState.

src/agent/utils/publisher_agent.py

The PublisherAgent compiles all generated content into a final document and publishes it in multiple formats (Markdown, PDF, Word, Azure Blob Storage). generate_full_content assembles the final text from outline_text, section_texts, statblocks, and items; write_report_by_formats handles file output; the published location is stored in OutlineState.url.

src/agent/utils/writer_agent.py

A generic agent that converts structured JSON content into full markdown text via LLMs, using predefined markdown examples selected through the MarkdownExampleKey enum (get_example_markdown, run_writing_text).

Data Flow

The system primarily uses an internal, stateful graph (StateGraph) for communication between its agent components. Each agent reads from and writes to the shared OutlineState as the adventure progresses from idea to published document.

  • External communication with LLMs (OpenAI) is handled via langchain-openai and ChatPromptTemplate for structured input/output.
  • External data retrieval is performed through TavilySearch and WikipediaRetriever tools.
  • File storage and retrieval leverage aiofiles for local file system operations and azure-storage-blob for cloud storage.

Cross-Cutting Concerns
  • Authentication & Authorization: API keys for OpenAI, LangSmith, Tavily, and Azure Storage are managed via environment variables (.env file and Docker Compose). The system assumes pre-authenticated access to these services.
  • Error Handling: No centralized error handling mechanism beyond standard Python exceptions is visible; LLM calls often include retry logic implicitly handled by LangChain/LangGraph.
  • Logging & Monitoring: Basic logging uses Python's logging module plus custom print_agent_output and log functions in src/agent/utils/views.py for colored console output. LangSmith integration (pytest.mark.langsmith, LANGSMITH_API_KEY) provides tracing and monitoring.
  • Configuration: Managed through environment variables (API keys and service URIs), pyproject.toml (dependencies and tool settings), langgraph.json (graph deployment), and docker-compose.yml (service orchestration).
  • Security: Sensitive information like API keys is handled via environment variables, preventing hardcoding. File operations include UTF-8 encoding with error replacement. Generated content may contain sensitive information and should be handled carefully.
