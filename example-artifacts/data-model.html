Entity Breakdown

Data persistence is primarily handled through a PostgreSQL database, with Redis used for caching and session management. The application leverages Pydantic models to define the structure and validation of data, which are then processed within a LangGraph agent workflow.

AdventureOutline

Defines the high-level structure and synopsis of an entire adventure, including its name, number of chapters, and overall story.

Relationships: one-to-many with Chapter.

Additional Entities

The following supporting entities are also defined across the schema and state modules:

  • Spell, Feats, BossMechanic — character abilities and boss phases (statblock schema)
  • SubSection, SubSectionList — granular chapter content blocks (section schema)
  • RelatedSubjects, Editor, Perspectives — editor-selection models
  • Summary, AnswerWithCitations, AnswerWithoutCitations, Queries — interview models
  • StatBlockList — collection wrapper for statblocks
  • InputState, OutlineState, OutputState, SectionImproverState, SectionTextState, Configuration — LangGraph state definitions
